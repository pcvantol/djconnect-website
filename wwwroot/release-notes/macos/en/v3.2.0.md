# DJConnect App 3.2.0

## Highlights

- Updated the Apple clients to the DJConnect 3.2 protocol line.
- iOS and macOS now use Home Assistant local first, remote fallback second, and offline state when neither route is reachable.
- Apple clients no longer expose a client-hosted `/api/device/*` API, Bonjour `_djconnect._tcp` service, or Client address pairing UI.
- Apple Watch now runs Home Assistant traffic through the paired iPhone for pairing, status, Ask DJ history, playback actions, voice, and push registration.
- Added Music Assistant/Spotify Direct backend summary support with backend-aware action forwarding.

## Notes

- Pairing still starts locally through Home Assistant.
- Watch users should open DJConnect on iPhone when the Watch says the iPhone is not reachable.
