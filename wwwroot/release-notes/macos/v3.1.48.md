# DJConnect App 3.1.48

### Added

- Added app, status, and pairing fields to the local device API so Home Assistant can show iPhone and Watch sensors with concrete values instead of unknown placeholders.

### Changed

- Apple Watch Now Playing refresh now shows a busy state, prevents duplicate refreshes, and temporarily disables transport and volume controls while the status request is running.
- Ask DJ speaker/output responses now render as list rows with a right-aligned activate button, matching playlist and album action cards.
- Local pairing now prefers usable LAN IPv4 addresses and hides unusable Watch fallback addresses when no reachable local network address is available.
- Backend collection refresh throttling now updates only after at least one devices, queue, or playlist refresh succeeds.

### Fixed

- Fixed Dutch What's New publication by adding localized release-note source files for this release so the public workflow no longer falls back to English content for `nl` URLs.
